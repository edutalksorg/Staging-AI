## Design Guidelines

### Color Palette
- **Primary**: #3B82F6 (Blue)
- **Secondary**: #8B5CF6 (Purple)
- **Success**: #10B981 (Green)
- **Warning**: #F59E0B (Orange)
- **Danger**: #EF4444 (Red)
- **Neutral**: #64748B (Slate)

### Fonts
- **Headings**: Montserrat Bold or Poppins Bold
- **Body**: Inter Regular or Roboto Regular
- **Code**: Fira Code or Consolas

### Icons
- Use Lucide React icons or Font Awesome
- Consistent style throughout
- Appropriate size (not too large or small)

### Images
- High resolution (at least 1920x1080)
- Professional quality
- Relevant to content
- Properly attributed if stock photos

### Animations
- Subtle and professional
- Not distracting
- Consistent timing (0.3-0.5 seconds)
- Use sparingly

## Presentation Checklist

### Before Creating Slides
- [ ] Understand audience (technical level, interests)
- [ ] Define key messages
- [ ] Gather all content and data
- [ ] Collect screenshots and images
- [ ] Prepare demo environment

### While Creating Slides
- [ ] Follow consistent design
- [ ] Use high-quality images
- [ ] Proofread all text
- [ ] Check font sizes (readable from back)
- [ ] Test animations
- [ ] Add slide numbers
- [ ] Include source citations

### Before Presentation
- [ ] Practice full presentation 3+ times
- [ ] Time each section
- [ ] Test on presentation computer
- [ ] Check projector compatibility
- [ ] Prepare backup (USB, cloud)
- [ ] Print handouts
- [ ] Charge laptop

### Day of Presentation
- [ ] Arrive 30 minutes early
- [ ] Test equipment
- [ ] Set up demo accounts
- [ ] Have water available
- [ ] Turn off notifications
- [ ] Set phone to silent
- [ ] Have backup plan ready
